# AIニュースDJ MVP設計書

## コンセプト
キャラクター性のある語り口でニュースを届けるパーソナルAIニュースアプリ。
ただの要約ではなく、「DJキャラ」がニュースを自分の言葉で語る。

## ターゲットユーザー
- 毎朝のニュースチェックを楽しくしたい人
- 堅いニュースを読むのが面倒な人
- AIツールに興味がある層

---

## コーディング方針（最重要）

**スピードより保守性・変更容易性を最優先する。**

### 基本原則
1. **関心の分離**：データ取得・AI処理・表示・音声はそれぞれ独立したモジュールに分ける。1ファイルに複数の責務を混ぜない。
2. **設定の外出し**：RSSフィードURL、DJキャラのプロンプト、ジャンル定義などは専用の設定ファイル（`config/`）にまとめる。コード本体を触らず設定だけで変更できるようにする。
3. **型定義の徹底**：TypeScriptを使い、データ構造は`types/`に型定義ファイルを置く。anyは使わない。
4. **コメントは「なぜ」を書く**：何をしているかはコードを読めばわかる。なぜその実装にしたかをコメントで残す。
5. **エラーハンドリング**：API呼び出し・RSS取得など外部通信は必ずtry-catchで囲み、失敗時の挙動を明示する。
6. **環境変数のバリデーション**：起動時に必須環境変数の存在チェックを行い、不足時は明確なエラーメッセージを出す。

│       └── news/

├── components/           # UIコンポーネント（表示専用、ロジックを持たない）
│   ├── NewsCard.tsx

│   ├── ai/

│   ├── storage/

### 変更シナリオ別の対応箇所
| やりたいこと | 触るファイル |
|-------------|-------------|
| DJキャラを追加 | `config/characters.ts` に1エントリ追加するだけ |
| RSSフィードを追加・変更 | `config/rss-feeds.ts` を編集するだけ |
| ジャンルを追加 | `config/genres.ts` に追加 |
| UIデザインを変更 | `components/` 内のコンポーネントのみ |
| AI要約ロジックを変更 | `lib/ai/summarizer.ts` のみ |
| TTSを外部APIに差替え | `lib/tts/speaker.ts` のインターフェースを維持して中身だけ差替え |
| 更新頻度を変更 | cron-job.orgの設定のみ（コード変更不要） |

## 技術スタック

| 項目 | 選定 |
|------|------|
| フレームワーク | Next.js (App Router) |
| ホスティング | Vercel (無料プラン) |
| AI API | Anthropic API (Claude) |
| 音声合成 | Web Speech API (ブラウザ内蔵・無料) |
| ニュースソース | RSSフィード |
| 定期実行 | cron-job.org (POST `/api/cron`、Bearer Token認証) |
| データ保存 | Vercel Blob Storage |

### 環境変数（Vercelダッシュボードで手動設定）
- `ANTHROPIC_API_KEY`
- `BLOB_READ_WRITE_TOKEN`
- `CRON_SECRET`

**重要：環境変数は絶対にClaude Codeから設定しない。必ずVercelダッシュボードから手動設定。**

## MVP機能一覧

### 1. ニュース収集（バックエンド）
- RSSフィードから複数ジャンルのニュースを自動取得
- 対応ジャンル（MVP時点）：
  - テクノロジー・AI
  - ビジネス・経済
  - 国内ニュース（総合）
  - エンタメ
  - スポーツ
- 各ジャンルに対応するRSSフィードURL（NHK、Yahoo!ニュース、TechCrunch Japan等）をコード内で管理
- cron-job.orgで毎日朝（日本時間7:00 = UTC 22:00）に実行

### 2. AI要約＋キャラ変換（バックエンド）
- 収集したニュースをAnthropic APIに送信
- 選択されたDJキャラのシステムプロンプトに基づいて要約・語り口を変換
- 1回の配信で5〜8本のニュースをピックアップ
- 変換結果をVercel Blobに保存

### 3. DJキャラクター（プリセット2〜3種）
以下はMVP時点のプリセット案（実装時に調整可）：

**キャラA：毒舌アナリスト**
- 皮肉混じりにニュースの裏を読む
- 「要するにこういうことでしょ」系の切り口

**キャラB：やさしい解説者**
- 難しいニュースを中学生でもわかるように説明
- 「これ大事なポイントなんだけどね」系

**キャラC：テンション高めのラジオDJ**
- 朝のFMラジオ風の明るいトーン
- 「おはようございます！今日のトップニュースいっちゃいましょう！」系

#### キャラ設計の拡張性
- キャラはシステムプロンプトのテンプレートとして管理
- 後からキャラ追加が容易な構造にする
- 将来的にはユーザーがカスタムキャラを作成できるUIを追加予定

### 4. フロントエンド（表示・操作）
- **レスポンシブデザイン**：スマホ・PC両対応
- **メイン画面**：
  - 今日のニュース一覧（DJキャラの語り口で表示）
  - キャラ切替ボタン（キャラを変えると語り口が変わる）
  - ジャンルフィルター（表示するジャンルを選択）
  - 再生ボタン（TTS音声で読み上げ）
- **設定画面**：
  - 購読ジャンルの選択
  - デフォルトDJキャラの選択

### 5. 音声再生（TTS）
- Web Speech API（SpeechSynthesis）を使用
- ブラウザ内で完結、サーバーコスト不要
- 再生/停止/次のニュースへスキップ
- 日本語音声を使用
- 注意：Web Speech APIの音声品質はブラウザ・OS依存。MVPではこれで十分、将来的に外部TTS APIへの差し替えも可能な設計にする

## API設計

### `POST /api/cron`
- cron-job.orgから毎日呼び出される
- Bearer Token認証（`CRON_SECRET`）
- 処理：RSS取得 → Claude API要約 → Blob保存
- 全プリセットキャラ分の要約を生成して保存

### `GET /api/news`
- 保存済みの今日のニュースデータを返す
- クエリパラメータ：`character`（キャラID）、`genre`（ジャンル）
